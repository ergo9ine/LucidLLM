# LucidLLM 기능 로드맵

## 추론 및 모델

- [x] **스트리밍 응답** — 토큰 단위로 실시간 출력하여 응답 대기 시간 체감 개선
- [ ] **멀티모달 입력** — 이미지 첨부 후 Vision 모델로 분석 (LLaVA, Moondream 등)
- [ ] **사고 과정 시각화** — DeepSeek-R1 등 추론 모델의 내부 사고 과정(`<think>`)을 별도로 시각화
- [ ] **JSON 모드** — 모델의 출력을 강제로 유효한 JSON 스키마에 맞추도록 제어 (Structured Output)
- [ ] **로컬 Whisper STT** — Web Speech API 대신 Transformers.js 기반 Whisper 모델로 고품질 음성 인식
- [ ] **로컬 TTS (Text-to-Speech)** — SpeechT5, Bark 모델을 사용하여 자연스러운 음성 합성 (Web Speech API 대체)
- [ ] **실시간 번역 (Real-time Translation)** — NLLB, M2M100 등 전용 번역 모델을 통해 빠르고 정확한 오프라인 번역 제공
- [ ] **문서/이미지 텍스트 추출 (OCR)** — 비전 LLM 등을 활용하여 이미지나 PDF 문서에서 텍스트 및 구조 추출
- [ ] **모델 비교 모드** — 동일 프롬프트를 두 모델에 동시 전송하여 응답 품질 비교
- [ ] **양자화 자동 추천** — 사용자 기기의 VRAM/메모리에 맞는 최적 양자화 레벨 자동 제안
- [ ] **모델 프리셋** — 인기 모델 목록을 원클릭으로 다운로드할 수 있는 큐레이션 프리셋
- [ ] **RAG 지원** — 로컬 문서(PDF, TXT, MD)를 업로드하여 검색 증강 생성 수행
- [ ] **Function Calling** — 도구 사용 인터페이스로 계산기, 웹 검색 등 외부 기능 연동
- [ ] **Transformers.js 태스크 확장** — text-generation 외에 요약, 분류, zero-shot 등 파이프라인을 UI에서 선택 실행
- [ ] **로컬 임베딩 파이프라인** — Transformers.js `feature-extraction` 기반 임베딩 생성으로 대화 검색/RAG 품질 향상
- [ ] **로컬 리랭커 파이프라인** — Cross-Encoder 모델로 검색 결과를 재정렬해 RAG 응답 정확도 개선
- [ ] **모델 번들 무결성 검사** — ONNX 외 tokenizer/config/generation 파일 누락 여부를 활성화 전 사전 검증
- [ ] **모델 설명 가능성 (Explainability)** — 토큰/문장 기여도, attention·attribution 시각화 및 응답 근거 추적
- [ ] **스마트 모델 라우팅 및 폴백** — 지연시간·품질·비용 기준의 동적 라우팅 및 폴백 정책
- [ ] **서명된 모델 번들 & 출처 검증** — 디지털 서명으로 모델 무결성 및 출처 확인
- [ ] **온디맨드 정밀도 튜닝(실험용)** — FP16/INT8 변환과 성능‑정확도 비교 툴 제공

## 채팅 UX

- [ ] **Markdown 렌더링** — 코드 블록 구문 강조, 테이블, LaTeX 수식 렌더링
- [ ] **아티팩트 / 라이브 미리보기** — 생성된 HTML/SVG/React 코드를 별도 창에서 즉시 렌더링 (Claude Artifacts 스타일)
- [ ] **장기 기억 (Long-term Memory)** — 사용자의 선호도나 주요 사실을 벡터 DB(로컬)에 저장하여 세션 간 기억 유지
- [ ] **컨텍스트 압축 (Context Compression)** — 긴 대화 내용을 요약하여 컨텍스트 윈도우 효율화
- [ ] **메시지 편집/재생성** — 전송된 메시지 수정 후 재생성 기능
- [ ] **대화 분기(Fork)** — 특정 메시지 시점에서 대화를 분기하여 다른 방향으로 탐색
- [ ] **대화 검색 (Semantic Search)** — 키워드 매칭뿐 아니라 임베딩 모델(all-MiniLM 등)을 활용한 의미 기반 검색
- [x] **대화 내보내기** — JSON 형식으로 채팅 기록 다운로드 (추후 Markdown, TXT 지원 예정)
- [ ] **드래그 앤 드롭 파일 첨부** — 채팅 입력란에 파일을 드롭하여 컨텍스트로 첨부
- [ ] **음성 입력/출력** — Web Speech API를 활용한 STT 입력 및 TTS 응답 읽기
- [ ] **실시간 협업 세션** — WebRTC 기반의 다중 사용자 실시간 대화·편집 및 세션 공유
- [ ] **대화 태그·폴더·북마크** — 대화 관리(태그/폴더/즐겨찾기)로 대화량 많은 사용자 지원
- [ ] **인라인 주석·리뷰(코멘트)** — 메시지 단위 주석 및 협업형 리뷰 기능
- [ ] **코드 실행 셀 (Sandboxed)** — 안전한 샌드박스 환경에서 코드 블록 실행 및 출력 미리보기
- [ ] **응답 피드백 및 메타(좋아요/문제 신고)** — 모델 품질 개선을 위한 사용자 피드백 수집 UI

## 성능 및 인프라

- [ ] **WebGPU 메모리 모니터링** — GPU 메모리 사용량 실시간 표시 및 OOM 사전 경고
- [ ] **추론 벤치마크** — 모델별 tokens/sec, 첫 토큰 지연시간(TTFT), 메모리 사용량 측정 대시보드
- [ ] **백그라운드 다운로드** — 모델 다운로드 중에도 기존 모델로 채팅 계속 가능
- [ ] **SharedArrayBuffer 지원** — COOP/COEP 헤더 설정으로 멀티스레드 WASM 추론 활성화
- [ ] **IndexedDB 폴백** — OPFS 미지원 브라우저를 위한 IndexedDB 기반 모델 캐시 폴백
- [ ] **Transformers.js 워커 풀** — 단일 워커 대신 작업 큐/워커 풀로 다운로드·추론·전처리 병렬 처리
- [ ] **모델 워밍업 모드** — 앱 시작 시 선택 모델을 1회 프리런하여 첫 토큰 지연시간(TTFT) 단축
- [ ] **모델 스케줄러(메모리 기반 스왑/프리페치)** — 사용 패턴에 따른 모델 캐시 관리로 지연시간/메모리 최적화
- [ ] **자동 리소스 프로파일링** — 모델별 성능·메모리 프로파일 수집 및 최적화 제안
- [ ] **동적 배칭 및 오프로드** — 여러 요청 병합 처리 및 디바이스 간 오프로드 전략
